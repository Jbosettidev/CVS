Mini-Guia: Branching e Pull Request

Estratégias de Branching

Branching permite criar linhas de desenvolvimento independentes. Exemplo: usar branches para novas funcionalidades ou correções.

Fluxo de Pull Request

Pull Requests permitem revisar código antes de integrar à principal. Sempre peça revisão e resolva conflitos antes de aceitar.

Commits Atômicos

Faça commits pequenos e focados. Exemplo: 1 commit por funcionalidade ou bug resolvido.

Resolução de Conflitos

Conflitos aparecem quando dois trabalhos afetam a mesma linha de código. Resolva localmente, teste e faça push corrigido.

Revisão de Código

Outra pessoa revisa cada PR, corrige erros e sugere melhorias antes do merge.

Automação

Ferramentas automáticas (CI/CD) verificam testes automaticamente após cada push ou PR.
